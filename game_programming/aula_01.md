> [!WARNING]
> Regra prática de compatibilidade de versão
> A Unity segue esta lógica aproximada:
> **Mesmo patch** (ex.: 6000.3.6f1 → 6000.3.7f1) => normalmente seguro
> **Mesmo minor** (ex.: 6000.3.x → 6000.3.y) => geralmente seguro
> **Minor diferente** (ex.: 6000.3 → 6000.4) => pode quebrar
> **Tipo de release diferente** (`f`inal, `b`eta, `a`lpha) => risco alto
